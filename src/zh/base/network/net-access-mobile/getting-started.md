# 入门

> 基于 HTTP 现代标准 · 核于 2026-06

## 速查

- **网络规模**：PAN（个域/蓝牙）< LAN（局域/楼）< MAN（城域）< WAN（广域，互联网是最大 WAN）
- **接入网**：「最后一公里」，把家庭/企业接入 ISP；ISP 分 Tier-1/二级/本地，靠对等(peering)与转接(transit)互联
- **宽带**：拨号 → DSL（电话线/非对称）→ 光纤 FTTH（PON 无源/GPON→XGS-PON）；同轴 Cable、卫星 LEO（Starlink 低延迟）
- ⚠️ **bit vs Byte**：网速用 bit（Mbps），文件用 Byte（MB/s），`100 Mbps ≈ 12.5 MB/s`
- **蜂窝**：2G(GSM)→3G→4G(LTE)→5G；小区/基站/切换 handover；5G 三场景 eMBB/URLLC/mMTC、Sub-6GHz vs 毫米波
- **移动弱网**：相比有线，高延迟、大抖动、多丢包、带宽波动、信号切换断连、IP 易变
- **RRC 无线电状态机**：蜂窝从空闲到激活有唤醒延迟（数百 ms）+ 尾延迟耗电——「频繁小请求」在移动端特别糟
- **Network Information API**：`navigator.connection.effectiveType`/`saveData` 按网速自适应（仅 Chromium、需特性检测）
- **CDN**：就近缓存静态内容（边缘节点 PoP）；缓存命中直接返回、未命中回源；调度靠 DNS 或 **Anycast**
- **性能指标**：带宽（吞吐上限）、延迟/RTT（往返）、丢包率、抖动 jitter；延迟 = 传播+传输+处理+排队
- **延迟常是瓶颈**：带宽不够可加钱，延迟受光速物理限制；弱网优化围绕「减少请求与往返」

## 从最后一公里到前端性能

这一叶把视角从「网络内部」拉到「网络边缘」——数据怎么进出网络、移动设备面对什么、以及这些如何影响你的页面：

1. **怎么接入**：家庭/企业通过接入网（光纤、蜂窝等）连到 ISP，ISP 之间再互联成全球互联网。
2. **移动的特殊性**：蜂窝网络延迟高、信号波动、还有 RRC 状态机带来的唤醒延迟与耗电——移动端不能照搬桌面的网络假设。
3. **怎么变快**：CDN 把内容缓存到离用户最近的边缘节点；前端则通过减少请求、压缩、预连接、自适应加载来对抗延迟。

::: tip 延迟才是前端性能的隐形主角
很多人以为「网慢 = 带宽不够」，但对于加载几十个小资源的网页，**延迟（RTT）往往才是瓶颈**——每个请求都要至少一个往返，而往返时间受光速和物理距离限制，加带宽也救不了。这正是 CDN 就近、HTTP/2 多路复用、`preconnect` 提前建连这些优化的根本动机。
:::
