# 接入网与 LAN/WAN/MAN

> 基于 HTTP 现代标准 · 核于 2026-06

## 速查

- **按地理规模分类**：网络从小到大依次为 **PAN**（个域网，约 10 米内）→ **LAN**（局域网，一栋楼/校园内）→ **MAN**（城域网，一座城市）→ **WAN**（广域网，跨地域/跨国）；**互联网是最大的 WAN**。
- **PAN（Personal Area Network）**：个人随身设备间的短距连接，典型技术是**蓝牙**、NFC，范围通常不超过 10 米（耳机、键鼠、手环）。
- **LAN（Local Area Network）**：限定在小地理范围内的网络，「通常在同一栋建筑内」——家庭 Wi-Fi、办公室网络是最典型例子；**几乎总是用以太网、Wi-Fi 或两者混合**连接设备。
- **MAN（Metropolitan Area Network）**：覆盖一座城市的网络，规模介于 LAN 与 WAN 之间（城市光纤环网、有线电视城域网）。
- **WAN（Wide Area Network）**：「延伸到大地理范围」的网络，可跨区域、跨国，**本质是把许多 LAN 连接起来**；常用**租用线路、MPLS、分组交换**等技术。
- **接入网（Access Network）**：「把终端用户连到其直属服务商」的那段网络，俗称**「最后一公里」**——把家庭/企业接入 ISP。
- **核心网（Core Network）vs 接入网**：核心网「把各个本地服务商互联起来」（骨干/长途），接入网只管「用户↔服务商」这一段；二者职责不同。
- **互联网 = 网络的网络**：LAN 接入一个更大的网络（**自治系统 AS**），AS 再与其他大网互联——「**一个 ISP 就是一个 AS 的例子**」（Cloudflare）。
- **ISP 分层**：一级 ISP（Tier-1，全球骨干，彼此**对等**免费互通）→ 二级 ISP（区域，向 Tier-1 购买**转接**）→ 本地 ISP（接到你家的那家）。
- **对等（peering）vs 转接（transit）**：对等是两网平等互换各自流量（通常免费），转接是向上游付费换取「通往全网」的可达性。
- **边界**：本页只讲规模分类与接入/核心/ISP 概念；**具体宽带接入技术**（光纤/DSL/HFC）见 [宽带接入技术](./broadband)，**蜂窝移动网络**见下一叶。

## 一、按地理规模给网络分类

谈「网络」时，**规模**是最直观的分类维度——从随身的几米到横跨大洲的上万公里，每一档都有自己的典型技术与用途。Wikipedia 将其从小到大排成一条谱系，前端工程师只需记住主干四档：

| 类型 | 全称 | 典型范围 | 典型场景 / 技术 |
| --- | --- | --- | --- |
| **PAN** | Personal Area Network | ≤ 约 10 米 | 蓝牙耳机、NFC、键鼠（个人随身设备） |
| **LAN** | Local Area Network | 一栋楼 / 校园（约 1 km 内） | 家庭 Wi-Fi、办公室网（以太网 + Wi-Fi） |
| **MAN** | Metropolitan Area Network | 一座城市 | 城市光纤环网、有线电视城域网 |
| **WAN** | Wide Area Network | 跨地域 / 跨国 | 企业跨城专线、**互联网** |

::: tip 一句话记住四档
**「人—楼—城—天下」**：PAN 是**人**身边的几米，LAN 是一栋**楼**，MAN 是一座**城**，WAN 是**天下**（跨地域乃至全球）。中间还有 **CAN（校园网）** 这一档（多栋楼），介于 LAN 与 MAN 之间，了解即可。
:::

谱系两端还有更细的分档（纳米网络、NFC、BAN 体域网等），但日常技术讨论里出现频率极低，记住主干四档足矣。

### LAN：家里和办公室的那张网

Cloudflare 给 LAN 的定义很朴素：**「限定在小地理范围内、通常在同一栋建筑内的网络」**，家庭 Wi-Fi 与小型企业网就是最常见的 LAN。它的两个关键特征：

- **连接技术**：「LAN 几乎总是用**以太网、Wi-Fi 或两者混合**」——以太网靠网线做物理连接，Wi-Fi 用无线电波连入（详见 net-link-lan 叶的[以太网帧结构](../../net-link-lan/guide-line/ethernet-frame)与 [Wi-Fi/802.11](../../net-link-lan/guide-line/wifi)）。
- **汇聚点是路由器**：「多数 LAN 在一个中心点接入互联网——**路由器**」；更大的 LAN 还会加**交换机**做更高效的转发。

> 顺带澄清：**VLAN（虚拟 LAN）** 是用软件把同一张物理网划成两张逻辑网，只需一台路由器、一条线路即可隔离流量——它和「按规模分类」无关，详见 net-link-lan 叶的 [VLAN](../../net-link-lan/guide-line/vlan)。

### WAN：把许多 LAN 连起来

Cloudflare 对 WAN 的定义一针见血：**「WAN 是一堆相连的 LAN 的集合，是局域网构成的广域网络」**，大小不限，可达上万公里。Wikipedia 进一步点明：**「互联网可被视为一个 WAN」**，是这类网络中最大的一个。WAN 的实现技术与 LAN 截然不同——LAN 设备本地自管，WAN 则依赖**租用线路、分组/电路交换、MPLS、SD-WAN**等跨距离手段。

## 二、接入网：连接用户与服务商的「最后一公里」

把视角从「网内」拉到「网与网之间」，第一个要厘清的概念就是**接入网**。Wikipedia 的定义是：**「接入网是电信网络中、把终端用户连到其直属服务商的那一段」**。它就是你家光猫到 ISP 机房之间的那截网络，业界俗称**「最后一公里」（last mile）**——也是整条链路中最贴近用户、铺设成本最高的一段。

接入网内部还可再分两层：

- **馈线 / 分布网（feeder / distribution network）**：较粗的主干段，从 ISP 机房延伸到小区/楼栋。
- **引入 / 边缘网（drop / edge network）**：最末端的引入线，直连到每户的终端点。

::: info 接入网 vs 核心网：职责分工
Wikipedia 明确把接入网与**核心网（core network）**对立：
- **接入网**只管 **「用户 ↔ 服务商」** 这一段（最后一公里）。
- **核心网**则负责 **「把各个本地服务商互联起来」**，是长途/骨干部分。

打个比方：接入网是从你家到本地邮局的那条小路；核心网是邮局之间运送邮包的高速公路网。**前端能感知到的弱网、首字节延迟，绝大多数发生在接入网这一段**——它带宽最受限、最易拥塞。
:::

接入网的**具体技术**（光纤入户 FTTH、DSL、HFC 同轴、固定无线等）不在本页展开，下一页 [宽带接入技术](./broadband) 专门讲。

### 为什么前端要懂「接入网」这一段

接入网看似离前端很远，实则是**前端性能的物理天花板**：

- **带宽不对称**：家用接入网普遍**下行远大于上行**（如 1000M 下 / 30M 上），上传大文件、推流时上行先成瓶颈——做上传组件时要对此有预期。
- **共享与拥塞**：同一栋楼/小区往往共享上游容量，晚高峰时接入段拥塞会直接抬高 RTT 与丢包率，表现为「网页突然变慢」。
- **首字节延迟**：请求要先穿过这段最后一公里才进入骨干，**接入段的物理时延无法靠代码消除**——只能靠把内容推近用户（CDN）来缩短后续路径。

换句话说，前端能优化的是「请求发出后怎么走得更短、传得更省」，而**接入网决定了「起跑线在哪」**——理解它，才知道哪些慢是代码能救的、哪些是物理决定的。

## 三、互联网 = 网络的网络

```text
        [ 你家 LAN ]──┐
                      │  接入网（最后一公里）
                 [ 本地 ISP ]
                      │  转接 transit（付费）
                 [ 二级 ISP ]
                      │  转接 transit（付费）
   [ 一级 ISP ]══对等 peering══[ 一级 ISP ]   ← 全球骨干，彼此免费互通
        │                          │
   [ 另一国 ISP ]            [ 内容方/CDN ]
```

**「互联网就是网络的网络」**（Cloudflare）：你家这张小 LAN 接入一个更大的网络——一个**自治系统（AS, Autonomous System）**，AS 再与其他大 AS 互联，层层嵌套，最终让相隔上万公里、分属两张不同 LAN 的两台电脑能互相通信。Cloudflare 一句话点透二者关系：**「一个 ISP 就是一个 AS 的例子」**——AS 是路由层面的「拥有独立路由策略和一批 IP 的大网」这一技术身份，ISP 则是它的商业角色。

### ISP 的分层与互联

ISP 并非铁板一块，按规模和角色分三层：

| 层级 | 角色 | 与其他网的关系 |
| --- | --- | --- |
| **一级 ISP（Tier-1）** | 全球骨干运营商 | 彼此**对等（peering）**，无需向任何人付费即可到达全网 |
| **二级 ISP（区域）** | 区域/国家级 | 向 Tier-1 买**转接（transit）**，对同级可能对等 |
| **本地 ISP（接入）** | 直接接到你家的那家 | 向上游买转接，经营接入网最后一公里 |

两网互联只有两种基本姿势：

- **对等（peering）**：地位平等的两张网**直接互换各自客户的流量**，通常**不计费**——双方都省了向上游付转接费。
- **转接（transit）**：向上游 ISP **付费**，换取「通往整个互联网」的可达性；你家本地 ISP 必然要向上游买转接才能让你访问全球站点。

::: warning 对前端的现实意义
你的请求从浏览器出发，往往要穿越「本地 ISP → 区域 ISP →（可能经一级骨干）→ 内容方所在网络」这条多跳路径，**中间每一次跨网（尤其对等/转接的交界点）都可能成为延迟与丢包的来源**。这正是 **CDN** 把内容推到「离用户最近的网络边缘」来缩短路径的根本动因——详见本叶后续的 [CDN 网络原理](./cdn-principle)。
:::

## 小结

- **按规模四档记忆**：PAN（人，约 10 米）→ LAN（一栋楼）→ MAN（一座城）→ WAN（跨地域/全球），**互联网是最大的 WAN**；中间还有 CAN（校园网）这一档。
- **LAN** 限定在一栋楼内，几乎总用**以太网 + Wi-Fi**，在路由器这个中心点接入互联网；**WAN 本质是「许多 LAN 的集合」**，靠租用线路/MPLS/分组交换跨距离互联。
- **接入网 = 最后一公里**，「把终端用户连到其直属服务商」；它带宽最受限、最易拥塞，**前端感知的弱网多发生在这一段**。
- **接入网 vs 核心网**：接入网管「用户↔服务商」，核心网管「服务商↔服务商」（骨干长途），职责分工清晰。
- **互联网 = 网络的网络**：LAN 接入一个 **AS（自治系统）**，AS 层层互联成全球网；**「一个 ISP 就是一个 AS 的例子」**。
- **ISP 三层**：一级（Tier-1 骨干，彼此**对等**免费）→ 二级（区域，买**转接**）→ 本地（接你家）；**对等免费互换流量、转接付费换全网可达**。
