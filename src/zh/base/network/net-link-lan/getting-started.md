# 入门

> 基于 HTTP 现代标准 · 核于 2026-06

## 速查

- **链路层职责**：在同一局域网内、相邻节点间成帧传输 + 差错检测；管「单跳」，IP 管「端到端」
- **MAC 地址**：48 位 / 6 字节十六进制（`00:1A:2B:3C:4D:5E`）；前 24 位 OUI 厂商标识；全球唯一、烧录在网卡
- 广播 MAC `FF:FF:FF:FF:FF:FF`；MAC 单跳寻址（每跳重写）、IP 端到端不变
- **以太网帧**（IEEE 802.3）：前导码 + 目的/源 MAC + 类型 + 载荷 + FCS；最小 64B、最大 1518B、MTU 1500、FCS 用 CRC 只检错不纠错
- **交换机**（二层）：MAC 地址表**自学习**（看源 MAC 学、查目的 MAC 转）；未知目的/广播帧泛洪
- **冲突域**：交换机每端口一个；**广播域**：默认整个交换网络一个（VLAN 才能分割）
- 交换机 vs 集线器：智能转发/独享带宽/全双工 vs 傻广播/共享带宽/半双工
- **VLAN**：802.1Q 标签逻辑划分广播域；Access 端口属单一 VLAN、Trunk 承载多 VLAN；VLAN 间通信需三层设备
- **ARP**：把同网段 IP 解析为 MAC（广播请求「谁是 1.1.1.1」→ 单播应答）；有缓存；ARP 欺骗 → 中间人攻击
- **Wi-Fi**（IEEE 802.11）：SSID/AP；2.4G 穿透强、5G/6G 速率高；**CSMA/CA**（无线不能边发边听，故「避免」冲突）；WPA3、Wi-Fi 7

## 链路层在做什么

网络层（IP）负责把数据包从源主机「端到端」送到目标主机，但每一跳的实际传输——「从这台路由器到下一台」「从你的电脑到家里的路由器」——都由**链路层**完成。它干三件事：

1. **成帧**：把网络层的 IP 包装进一个「帧」，加上目的/源 MAC、差错校验。
2. **MAC 寻址**：用 48 位 MAC 地址标识同一局域网内的设备（IP 是逻辑地址、跨网络；MAC 是物理地址、管单跳）。
3. **差错检测**：用帧尾的 FCS（CRC 校验）发现传输出错的帧并丢弃（不纠错，重传交给上层）。

```
IP 包 ──封装──▶ [前导码|目的MAC|源MAC|类型|IP包(载荷)|FCS] ──▶ 比特上线路
                同一局域网内：交换机按目的 MAC 转发
                跨网段：先 ARP 查网关 MAC，帧发给网关（路由器）
```

::: tip MAC 与 IP：管单跳 vs 管端到端
一个数据包从你的电脑到远端服务器，**源/目的 IP 始终不变**（端到端标识），但**目的 MAC 每过一个路由器就重写一次**（下一跳标识）。MAC 像「下一个路口往哪拐」，IP 像「最终目的地」。
:::

## 一个高频混淆：ARP ≠ DNS

两者都是「解析」，但层级完全不同：

- **DNS**：把**域名**解析为 **IP**（应用层，跨网络）——见「DNS 域名系统」叶。
- **ARP**：把同网段的 **IP** 解析为 **MAC**（链路层，本地单跳）——因为帧必须填目的 MAC，而上层只给了 IP。
