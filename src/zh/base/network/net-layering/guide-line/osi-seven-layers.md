# OSI 七层逐层职责

> 基于 HTTP 现代标准 · 核于 2026-06

## 速查

- **OSI 七层（自下而上）**：物理层 → 数据链路层 → 网络层 → 传输层 → 会话层 → 表示层 → 应用层（层 1 到层 7）。
- **OSI 是「参考模型」不是实现**：它是 ISO 提出的抽象概念模型，用来描述与排障；现代互联网实际跑的是更简单的 TCP/IP 协议族（详见下一页）。
- **各层数据单元（PDU）**：物理层=比特（bit）、数据链路层=帧（frame）、网络层=包（packet）、传输层=段（segment，UDP 习惯叫数据报 datagram），会话层及以上统称数据（data）。
- **物理层**：传输原始比特流（电/光/电磁信号），管线缆、网卡、信号编码；设备有网卡、集线器（hub）。
- **数据链路层**：在同一网络内的相邻节点间成帧传输，用 **MAC 地址**寻址，做帧内的差错与流量控制；设备有交换机（switch）、网桥。
- **网络层**：跨网络转发，用 **IP 地址**逻辑寻址并选最优路径（路由 routing）；协议有 IP / ICMP / IGMP / IPsec，设备是路由器（router）。
- **传输层**：端到端通信，把数据切成段并在对端重组，负责流量控制与差错恢复；**TCP**（可靠、面向连接）与 **UDP**（快、无连接）靠**端口号**区分应用。
- **会话层**：建立、维持、关闭通信会话，并可用检查点（checkpoint）支持断点续传。
- **表示层**：数据的「翻译官」——负责编码转换、加密/解密、压缩/解压，让应用层拿到可直接消费的数据。
- **应用层**：唯一直接面向用户数据的层，承载 **HTTP / DNS / SMTP / FTP / DHCP** 等应用协议；注意浏览器、邮件客户端本身不属于该层。
- **记忆口诀**：从层 1 到层 7 取首字母 **P**hysical–**D**ata link–**N**etwork–**T**ransport–**S**ession–**P**resentation–**A**pplication，背英文助记句 **「Please Do Not Throw Sausage Pizza Away」**。

## 一张总表看全七层

OSI（Open Systems Interconnection，开放系统互连）参考模型把通信系统拆成**七个相互堆叠的抽象层**，每层只干一件事，并只与紧邻的上下层打交道。先用一张总表建立全局印象：

| 层 | 名称 | 核心职责 | 数据单元（PDU） | 典型协议 | 典型设备 |
| --- | --- | --- | --- | --- | --- |
| 7 | 应用层 Application | 面向用户的应用协议与数据 | 数据 Data | HTTP、DNS、SMTP、FTP、DHCP | 浏览器、邮件客户端（软件本身不属此层） |
| 6 | 表示层 Presentation | 编码转换、加密、压缩 | 数据 Data | TLS、MIME、ASCII | 加密设备 |
| 5 | 会话层 Session | 建立 / 维持 / 关闭会话 | 数据 Data | RPC、NetBIOS、PPTP、SOCKS | 应用服务器 |
| 4 | 传输层 Transport | 端到端、分段重组、流控差错 | 段 Segment（UDP 称数据报） | TCP、UDP、QUIC、SCTP | 网关 |
| 3 | 网络层 Network | 逻辑寻址（IP）与路由选路 | 包 Packet | IP、ICMP、IGMP、IPsec | 路由器 |
| 2 | 数据链路层 Data Link | 同网相邻节点成帧、MAC 寻址 | 帧 Frame | Ethernet、Wi-Fi(802.11)、PPP、ARP | 交换机、网桥 |
| 1 | 物理层 Physical | 比特流的物理传输 | 比特 Bit | Ethernet、USB、Bluetooth、RS-232 | 网卡、集线器、中继器 |

::: tip 助记：层号与「层 4 是分水岭」
口诀句 **「Please Do Not Throw Sausage Pizza Away」**对应层 1→7 的首字母。再记一条规律：**层 4 及以上（传输层、会话层、表示层、应用层）是「端到端」的主机职责，层 3 及以下（网络层、数据链路层、物理层）才是网络设备真正参与转发的层**——这条线在排障时极有用。
:::

## 逐层职责（自下而上）

理解 OSI 最顺的方向是**自下而上**：从最贴近硬件的比特，一路抽象到用户能读懂的数据。

### 层 1 · 物理层（Physical）

负责在物理介质上**收发原始比特流**。它不关心比特代表什么含义，只管把 `1` 和 `0` 变成可在介质上传播的信号——铜缆里的电压、光纤里的光脉冲、Wi-Fi 里的电磁波，并约定收发双方如何区分 `1` 与 `0`（信号编码与时钟）。

- **数据单元**：比特（bit）。
- **关键词**：线缆、接口、电平、调制、网卡的物理收发部分。
- **典型设备**：网卡（物理部分）、集线器（hub）、中继器（repeater）。

### 层 2 · 数据链路层（Data Link）

负责在**同一个网络内**、由物理层直连的**相邻节点之间**可靠地传输数据。它把网络层下来的包封装成**帧（frame）**，用 **MAC 地址**（网卡硬件地址）寻址，并在本段链路上做差错检测与流量控制。

- **数据单元**：帧（frame）。
- **关键词**：MAC 地址、成帧、CRC 校验、局域网内寻址。
- **典型协议 / 设备**：Ethernet、Wi-Fi(802.11)、PPP、ARP；交换机（switch）、网桥（bridge）。

::: info 数据链路层 vs 网络层：同网 vs 跨网
两者都做寻址与转发，区别在范围：**数据链路层只管同一网络内（用 MAC）**，**网络层负责跨越不同网络（用 IP）**。交换机工作在层 2 看 MAC，路由器工作在层 3 看 IP——这正是「二层交换、三层路由」的由来。
:::

### 层 3 · 网络层（Network）

负责在**不同网络之间**转发数据：把传输层的段切成**包（packet）**，用 **IP 地址**做逻辑寻址，并为每个包在错综复杂的网络中**选出最优路径（路由 routing）**。若通信双方在同一网络内，理论上不需要网络层。

- **数据单元**：包（packet）。
- **关键词**：IP 地址、路由选路、跨网转发。
- **典型协议 / 设备**：IP、ICMP、IGMP、IPsec；路由器（router）。

### 层 4 · 传输层（Transport）

负责两台主机间的**端到端通信**。它把会话层的数据切成**段（segment）**交给网络层，并在接收端重组；同时承担**流量控制**（避免快的发送方淹没慢的接收方）与**差错控制**（校验完整性、必要时请求重传）。用**端口号**把数据交付给正确的应用进程。

- **数据单元**：段（segment）；UDP 习惯称数据报（datagram）。
- **两大协议**：**TCP**——面向连接、可靠、有序、有重传；**UDP**——无连接、不保证可靠，但开销小、延迟低。
- **关键词**：端口号、可靠性、流控、连接管理。

::: warning 「端到端」是传输层的关键词
网络层只保证「把包送到对的主机」，但**「确保数据完整、有序、不丢」是传输层（TCP）的事**。HTTP 之所以能稳定收发整段报文，正是站在 TCP 的可靠传输之上——这也是为什么 HTTPS 的 TLS 通常被画在「传输层之上、应用层之下」。
:::

### 层 5 · 会话层（Session）

负责**建立、维持与关闭**两端之间的通信会话。会话指从开启到关闭这段持续交互的时间窗口；会话层确保它开得够久以传完全部数据，又能及时关闭以释放资源。它还能用**检查点（checkpoint）**做同步——传大文件中途断连，可从最近检查点续传，而非从头再来。

- **数据单元**：数据（data）。
- **关键词**：会话生命周期、同步、检查点 / 断点续传。

### 层 6 · 表示层（Presentation）

数据的**「翻译官」**，负责让数据「可被应用层消费」。三大职责：**编码转换**（不同设备用不同编码时做语法翻译）、**加密 / 解密**（发送端加密、接收端解密，呈给应用层明文）、**压缩 / 解压**（减少传输量、提升效率）。

- **数据单元**：数据（data）。
- **关键词**：字符编码、加解密、压缩。
- **典型协议**：TLS、MIME、ASCII。

### 层 7 · 应用层（Application）

**唯一直接面向用户数据**的层。它承载用户应用所依赖的各种**应用协议与数据操作**，把有意义的数据呈现给用户。要点：浏览器、邮件客户端这些**软件本身并不属于应用层**，应用层指的是它们所依赖的协议（如 HTTP、SMTP）与数据处理。

- **数据单元**：数据（data）。
- **关键词**：应用协议、面向用户、资源访问。
- **典型协议**：HTTP、DNS、SMTP、FTP、DHCP。

## OSI 是「参考模型」，不是实际实现

这是最容易被忽视、却最重要的一点：**OSI 是一套抽象的概念 / 参考模型**，由 ISO 制定，用来给「不同系统如何通信」提供统一语言与协调标准的基础——它更像一把**用来描述和排障的标尺**，而非现网真正运行的协议栈。

- **现代互联网并不严格遵循 OSI**，而是更贴近**更简单的 TCP/IP 协议族**。历史上 OSI 自家那套协议被普遍认为过于复杂、低效、难以落地，最终是互联网协议族成了事实标准。
- **它的价值在排障**：把网络问题「分层定位」——一个人上不了网，还是上千用户访问某站点失败，都能借分层把问题缩到某一层，省去大量无谓排查（应用层故障查层 7，连通性问题查层 3/4）。

::: tip 学 OSI 的正确姿势
不必纠结「现实里会话层、表示层是不是独立存在」——在 TCP/IP 世界里它们大多被并入应用层。把 OSI 当成**一套通用词汇与分层心智**来用：它让你能精准地说出「这是二层问题还是三层问题」「TLS 大致在表示层附近」，沟通与排障效率立竿见影。
:::

## 小结

- OSI 七层**自下而上**为：物理层 → 数据链路层 → 网络层 → 传输层 → 会话层 → 表示层 → 应用层（层 1→7）。
- 各层数据单元（PDU）：**比特 → 帧 → 包 → 段 →（数据 / 数据 / 数据）**；寻址关键词：数据链路层看 **MAC**、网络层看 **IP**、传输层看**端口**。
- 设备归层：集线器（层 1）、交换机（层 2）、路由器（层 3）；协议归层：IP（层 3）、TCP/UDP（层 4）、TLS（层 6 附近）、HTTP/DNS（层 7）。
- 记忆口诀 **「Please Do Not Throw Sausage Pizza Away」**对应层 1→7 首字母。
- **OSI 是抽象的参考模型，不是实际实现**；现网更贴近 TCP/IP，OSI 的最大价值是「分层排障」的通用语言。
