# 抖音小程序 vs 微信小程序

> 基于抖音小程序（基础库 4.x）· 核于 2026-07

## 速查

- **语法大面积一致**（双线程、`setData`、数据绑定 <code v-pre>{{ }}</code>、同名生命周期），差异在**前缀 + 生态 + 战略**
- 后缀 `.ttml`/`.ttss`/`.sjs` vs `.wxml`/`.wxss`/`.wxs`；API `tt.*` vs `wx.*`；指令 `tt:` vs `wx:`
- 宿主：抖音/今日头条（+2 极速版） vs 微信
- 流量逻辑：**内容公域·算法推荐·即时转化** vs **社交私域·分享裂变·长期留存**
- 主包体积上限约 **4MB**（较宽松） vs 微信 2MB；注册**以企业为主**（个人受限） vs 微信个人/企业皆可
- 支付 `tt.pay`（担保支付） vs `wx.requestPayment`（微信支付）
- **易错**：语法相似 ≠ 复制即用——支付/登录/分享/音视频/用户信息等生态 API 必须逐个适配

## 一、全面差异对照

| 维度 | 抖音（字节） | 微信 |
| --- | --- | --- |
| 文件后缀 | `.ttml` / `.ttss` / `.sjs` | `.wxml` / `.wxss` / `.wxs` |
| API 前缀 | `tt.*` | `wx.*` |
| 指令前缀 | `tt:if` / `tt:for` | `wx:if` / `wx:for` |
| 宿主 App | 抖音 / 今日头条 / 两个极速版 | 微信 |
| 流量逻辑 | 内容公域·算法推荐·即时转化 | 社交私域·分享裂变·长期留存 |
| 主包体积上限 | 约 4MB（较宽松） | 2MB |
| 分包上限 | 约 20MB（社区口径，以官方为准） | 单个 2MB / 总 20MB |
| 注册主体 | **以企业为主**，部分行业定向邀请；个人受限 | 个人 / 企业皆可（个人不能开支付） |
| 支付 | `tt.pay`（担保支付，需入驻） | `wx.requestPayment`（微信支付） |
| 登录 | `tt.login`→`code`→`code2session`（流程同构） | `wx.login`→`code`→`code2Session` |
| 特色 | 短视频挂载/直播小风车/`aweme-data`/音视频强 | 社交裂变/公众号/生态最全 |
| 开发者工具 | 抖音开发者工具 | 微信开发者工具 |

## 二、迁移：不能只换前缀

官方「搬家工具」/ `wx2tt` 可批量把后缀 `wxml/wxss`→`ttml/ttss`、前缀 `wx.`→`tt.`、`wx:`→`tt:`，但：

- **生态强绑定 API 必须逐个手动适配**：支付（担保支付模式）、登录（抖音登录态）、分享、音视频、用户信息授权。
- 部分 API「只支持抖音宿主」，多宿主需判端降级。
- 结论：**语法相似 ≠ 复制即用**。真正的多端一致更推荐用 **Taro / uni-app**（把抖音作为一个输出端）。

## 三、战略/生态对比（考点）

- **微信 = 社交私域**：入口是分享、群、公众号；靠社交关系裂变、长期留存。
- **抖音 = 内容公域**：入口是短视频挂载、直播小风车、评论锚点；靠算法推荐把公域流量即时转化。
- 选型直觉：做**内容/直播带货/即时转化**型业务，抖音的公域入口是天然优势；做**社交/工具/长期复访**型，微信生态更成熟。
