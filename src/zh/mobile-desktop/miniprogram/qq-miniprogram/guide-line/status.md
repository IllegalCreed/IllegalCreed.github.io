# QQ小程序 现状与定位

> 基于 QQ小程序 · 核于 2026-07

## 速查

- **基础库冻结**：官方 changelog 最新条目 **v1.60.0（2022-12-22）**，之后无新版本记录——平台事实上**低维护**
- **定位**：微信小程序的**同源近亲分支**，现已**边缘化**；出题/学习重点在「与微信的关系与差异」，而非当活跃主力
- **AI 计划归属纠偏**：2026「AI 小程序成长计划」「微信 AI Agent / 小微」均属**微信**，**不是 QQ**——勿张冠李戴
- **是否停止新注册**：仅一处搜索摘要提「维护中/暂停新建」，无官方硬证据→**待核**，不宜写死为「已停注册」
- 开放注册时间：2019-07-23

## 一、维护现状：事实上的低维护

判断 QQ 小程序活跃度，最硬的一手信号是**基础库更新日志**：

- 官方 `changelog` 最新条目停在 **v1.60.0（2022-12-22）**，2023 至 2026 无新版本记录。
- 官方文档站仍在线（q.qq.com/wiki 可读），但内容多为 2019–2022 旧文，鲜有新动作。
- 结合腾讯 2026 年的小程序投入几乎全在微信侧，可判断 QQ 小程序处于**存量维护 / 边缘化**状态。

> 因此本叶把 QQ 小程序定位为「微信小程序的同源分支、现已边缘化」，学习价值在于**理解它与微信的同源关系与命名差异**，而非把它当作 2026 年的活跃主力平台。

## 二、纠偏：AI 小程序计划是微信、不是 QQ

2026 年围绕「小程序 + AI」的动作容易张冠李戴，务必分清：

- **「AI 小程序成长计划」**（激励期 2026 全年、送云开发资源 + 混元 Token 额度）——属**微信小程序**生态。
- **微信内置 AI Agent /「小微」总控台**——同样是**微信**侧。
- **QQ 小程序**未见对应的 AI 计划或大规模生态推广。

把上述 AI 动作记成 QQ 的，是常见错误。

## 三、是否停止新注册（存疑，勿写死）

- 一处搜索摘要提到 QQ 开放平台「维护中、暂不支持新建小游戏/小程序」，但**英文/官方源无法证实**「已停止注册 / 已下线」这一强论断。
- 结合 changelog 冻结，可稳妥表述为「**事实上低维护**」；但「已停止注册」应标为**待核**，不作定论。

## 四、选型建议

- 若目标用户主要在 QQ，且能接受低维护现状，可做——但要预期官方支持有限、生态资源少。
- 需要覆盖 QQ 的团队，通常直接用 **uni-app / Taro** 等跨端框架把 QQ 作为一个输出目标端，而非单独为 QQ 维护一套工程（详见[与微信的关系与差异](./vs-wechat)与[登录与迁移](./login-migration)）。
